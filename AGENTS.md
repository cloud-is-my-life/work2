# AGENTS.md (짧은 작업 기준서)

## 0) 채점 환경 전제 (최우선)

- 이 저장소 과제는 **CloudShell의 AWS CLI** 또는 **AWS Management Console** 기준으로 수행/채점된다고 가정한다.
- 따라서 모든 레퍼런스/예시는 아래 원칙을 따른다.
  1. **CloudShell 복붙 가능 명령**
  2. **Console에서 동일하게 재현 가능한 절차**
  3. 로컬 전용 도구/환경(커스텀 스크립트, 로컬 파일 의존) 가정 금지

---

## 1) 문서/과제 작성 원칙

- 쓸데없는 이론 제외, **실행 순서 + 함정 + 검증 명령** 중심.
- placeholder는 반드시 대문자(`ACCOUNT_ID`, `DATA_BUCKET`)로 명확히 표기.
- 명령/정책/DDL은 “복붙 즉시 실행” 형태 유지.
- 가능한 경우 **CLI 방법 + Console 위치**를 같이 제공.

---

## 2) 과제 제작 체크리스트 (짧게)

- [ ] CloudShell CLI로 그대로 실행 가능한가?
- [ ] Console로도 같은 결과를 만들 수 있는가?
- [ ] IAM/KMS/S3 권한 누락 포인트를 명시했는가?
- [ ] “성공 확인 쿼리/명령”이 포함됐는가?
- [ ] Projection/CTAS/SerDe 함정을 눈에 띄게 표시했는가?

---

## 3) 채점 규정 (직종설명서)

- [ ] 경기 종료 이후, 선수는 채점지에 명시되지 않은 인프라에 영향이 가는 Action을 취할 수 없다. 
- [ ] 채점 스크립트는 언제 실행하더라도 결과가 동일하도록 멱등성을 보장해야 한다.
- [ ] 1과제의 경우 Bastion host를 통해, 2과제의 경우 Cloudshell을 사용하여 채점한다. 
- [ ] 필요 시 AWS Management Console을 사용하여 채점할 수 있으나, 이 경우 선수의 인프라에 영향이 가는 Action(ex 새로 Athena 쿼리 작성, ec2 생성 등)은 지양 또는 하지 않도록 한다.
