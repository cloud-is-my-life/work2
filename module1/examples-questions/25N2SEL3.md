No 3. Secure access to EFS
Module: 12) File System security
해당 과제 풀이는 아일랜드(eu-west-1) 리전에서 진행합니다. 
인스턴스의 권한에 따라 EFS 디렉토리 접근이 제한되는 인프라를 구성합니다. 
채점 시 Bastion 호스트 및 App 1 인스턴스를 사용하므로, 해당 인스턴스들에는 권한 또는 접근 관련 이슈가 발생하지 않도록 주의해야 합니다. 문제 지문에서 중괄호로 감싸진 부분은 선수가 직접 설정해야 하는 변수입니다. 채점 시 sshpass 라이브러리를 사용하므로, Bastion Host 및 App 1 인스턴스에 설치해야 합니다. 해당 라이브러리는 sudo yum install -y sshpass 명령어를 통해 설치합니다.

1. VPC
AWS VPC를 이용해 Public, Private 서브넷을 두 쌍 구성합니다. Subnet Name, CIDR 값은 아래 표를 참고합니다. Public 서브넷은 igw를 통해 인터넷에서 직접 접근할 수 있어야 하며, App 서브넷은 NAT G/W를 통해 접근할 수 있어야 합니다.

- VPC Name : efs-vpc
- VPC cidr : 10.128.0.0/16
- Subnet : efs-pub-b, efs-app-b, efs-pub-c, efs-app-c


2. IAM
wsi-ec2-efs-role이라는 이름을 가진 IAM Role을 생성합니다. 해당 역할은 EFS 스토리지에 대한 읽기, 쓰기 권한이 부여되어야 하며, 이보다 과한 권한이 부여되어선 안됩니다. 또한, EC2 인스턴스에 Attach 될 수 있어야 합니다.

3. EC2
EC2를 활용해 하나의 Bastion Host와 App 인스턴스 2대를 생성하고, 생성 시 내부 IP는 수동으로 설정하여야 합니다.

Bastion 호스트는 efs-pub-b 서브넷에 생성되어야 합니다. Bastion 호스트는 채점을 위해 사용되므로 권한 또는 접근 관련 이슈가 없도록 주의해야 합니다. 특히, 채점 과정에서 ip가 변경되는 일이 없게끔 인스턴스 재시작 시에도 IP는 변경되지 않도록 구성해야 합니다.

Bastion Host
- Name : efs-bastion
- IP : 10.128.0.199

App EC2는 private subnet 2개에 각각 1대씩 구성합니다. App ec2의 경우에도 채점에 사용될 수 있기 때문에 접속 및 권한 등의 문제가 없어야 하며, 비밀번호로 접속할 수 있어야 합니다.

App 1 / App 2
- Name : efs-app-1 / efs-app-2
- IP : 10.128.128.199 / 10.128.144.199
- OS : Amazon Linux 2023
- 비밀번호 : wsi{선수등번호}
- Tag는 AppRole: wsi-app 태그가 있어야 합니다.

4. KMS
wsi-kms라는 별칭을 가진 KMS 키를 생성합니다. EFS 및 wsi-ec2-efs-role IAM 역할에서 사용할 수 있도록 정책을 구성해야 합니다.

5. EFS 프로비저닝
App 인스턴스에서만 접근 가능한 EFS 파일 시스템을 구성해야 합니다. 이 때, Bastion 호스트에서는 마운트 또는 EFS 접근이 불가능해야 합니다. 모든 App EC2 인스턴스는 wsi-ec2-efs-role 역할을 통해 EFS에 대한 접근 권한을 획득해야 하며, 해당 역할이 없는 인스턴스는 EFS에 접근할 수 없어야 합니다. 정책 단에서 전송 과정에서의 암호화를 강제해야 합니다. 

 EFS는 아래 조건대로 구성합니다.
- Name : wsi-efs-fs
- KMS 키 : wsi-kms
- 탑재 대상 IP (각 가용구역별) : 10.128.128.111 / 10.128.144.111
- 기본적으로 액세스 포인트를 통해 /app/wsi{선수등번호} 하위 경로만 접근 가능해야 하며, 루트(/) 디렉토리에 대한 접근은 금지되어야 합니다.
- 마운트 가능 인스턴스는 가용구역 B 또는 C에서 생성된 인스턴스로 제한되어야 합니다.

아래 조건을 만족해야 EFS에 접근할 수 있도록 정책을 구성해야 합니다.
1. 접근 가능 기간은 2025년 9월 20일 UTC 3:00부터 2025년 9월 26일 UTC 18:00으로 제한되어야 합니다.
2. 접근하는 인스턴스에는 AppRole: wsi-app 태그가 포함되어 있어야 합니다.

EFS에 접근하기 위한 액세스 포인트는 아래와 같이 구성합니다.
- 파일 시스템 : wsi-efs-fs
- 액세스 포인트 이름 : wsi-efs-ap
- 루트 디렉터리 경로 : /app/wsi{선수등번호}




6. EFS 연결
2번에서 구성한 2개의 App 인스턴스에서 EFS를 마운트합니다. EFS 마운트는 인스턴스 재부팅 이후에도 유지되어야 하며, 마운트 지점은 /mnt/efs로 고정합니다. 또한, 항상 액세스 포인트를 이용해서 마운트해야 합니다.

연결한 후, /mnt/efs 경로에 “Hello from WorldSkills” 라는 내용을 가진 hello-{선수등번호}.txt 파일을 생성합니다. 이미 해당 파일이 존재한다면, 생성하지 않습니다.
